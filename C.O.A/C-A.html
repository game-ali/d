PART 1: Basic Structure & Functional Units

  • 1. What are the basic functional units of a computer?
    Input unit, memory unit, arithmetic and logic unit (ALU), output unit, control unit, and interconnection network.
  • 2. What is the function of the input unit?
    Accepts information from outside (keyboard, mouse) and sends it to the processor.
  • 3. Where is data stored after being received from the input unit?
    In the computer’s memory, for processing or later use.
  • 4. What specifies the operations to be performed on the stored data?
    A program, which is a sequence of instructions stored in memory.
  • 5. How are results delivered to the external world from the computer system?
    Through the output unit, using devices like monitor and printer.
  • 6. What is the role of the control unit?
    Directs and coordinates all operations by sending control signals.
  • 7. What is the role of the interconnection network?
    Allows all units to communicate and exchange data/signals.
  • 8. What distinguishes primary memory from secondary storage?
    Primary memory is fast and volatile (RAM); secondary storage is slower, non-volatile (hard disk).
  • 9. What is cache memory and why is it important?
    Small, fast memory that stores frequently used data to speed up execution.
  • 10. What is the function of the ALU in a computer?
    Performs arithmetic (add, subtract) and logic (AND, OR) operations.

PART 2: Registers, Data Path & Von Neumann Architecture

  • 1. What is the function of the Program Counter (PC) in a computer system?
    Holds the address of the next instruction to fetch.
  • 2. What role does the Instruction Register (IR) play during program execution?
    Holds the current instruction being executed.
  • 3. Explain the function of general-purpose registers (R0 to Rn−1).
    Temporarily store operands and results for quick access.
  • 4. What does the Memory Address Register (MAR) do?
    Holds the address for memory access (read or write).
  • 5. What is stored in the Memory Data Register (MDR)?
    Holds the data being transferred to or from memory.
  • 6. Describe the purpose of the Accumulator register.
    Used to store intermediate results of arithmetic/logic operations.
  • 7. What is the function of the processor–memory interface?
    Controls data transfer between processor and memory (using control signals).
  • 8. Outline the typical steps involved in program execution.
    Load program, set PC, fetch instruction, decode, execute, store results, repeat.
  • 9. What is an interrupt and when does it occur?
    A signal for immediate processor attention, e.g., from hardware or urgent event.
  • 10. Define Von Neumann Architecture.
    Computer design where both data and instructions are stored in the same memory.

PART 3: Number Systems & Computer Arithmetic

  • 1. Define unsigned integers in binary. Give two examples.
    Binary numbers that are always positive. Examples: memory addresses, roll numbers.
  • 2. What is the main difference between signed and unsigned binary numbers?
    Signed can be negative or positive; unsigned are only positive.
  • 3. What does the Most Significant Bit (MSB) indicate in signed numbers?
    0 for positive, 1 for negative.
  • 4. List the three systems for representing signed binary numbers.
    Signed magnitude, 1’s complement, 2’s complement.
  • 5. How is a negative number represented in signed magnitude?
    Set MSB to 1. Example: +5 = 0101, -5 = 1101.
  • 6. How do you get the 1’s complement of a binary number?
    Flip each bit (0 to 1, 1 to 0).
  • 7. Why does two’s complement have a unique zero representation?
    Both +0 and -0 are 0000.
  • 8. What is the advantage of fixed-point representation?
    Fast and easy calculations.
  • 9. What are the three main fields in fixed-point numbers?
    Sign, integer, and fractional fields.
  • 10. What is normalization in floating point?
    Adjusting the number so only one non-zero digit is left of the point.

PART 4: Memory System Design & I/O

  • 1. What is RAM and what is its main feature?
    Random Access Memory; fast, volatile, used for temporary data storage.
  • 2. What is the main purpose of ROM?
    Read Only Memory; stores permanent data like BIOS.
  • 3. What is cache memory?
    A small, very fast memory between CPU and main memory for frequently accessed data.
  • 4. What is memory interleaving?
    Splitting memory into modules to allow simultaneous access and faster data fetching.
  • 5. What is direct mapping in cache?
    Each block of memory maps to only one possible cache location.
  • 6. What is associative mapping in cache?
    Any memory block can go anywhere in the cache.
  • 7. What is set-associative mapping?
    Cache is divided into sets; each memory block can go in any slot within a set.
  • 8. What is a cache hit and a cache miss?
    Hit: data found in cache; miss: data not found (must fetch from main memory).
  • 9. What is DMA (Direct Memory Access)?
    A method for devices to transfer data to/from memory without CPU involvement.
  • 10. What is the difference between isolated I/O and memory-mapped I/O?
    Isolated I/O has separate address space for I/O; memory-mapped I/O uses same space as memory.

PART 5: Pipelining and Parallel Processors

  • 1. What is pipelining in computer architecture?
    Pipelining is dividing a task into separate stages, with each stage handled by a different processor segment, so multiple instructions can be in progress at the same time.
  • 2. How does instruction pipelining increase CPU performance?
    It overlaps the execution of multiple instructions, so a new instruction can start before the previous one finishes, increasing throughput.
  • 3. What are the three main types of pipeline hazards?
    Data hazards (data dependencies), control hazards (branches/jumps), and structural hazards (hardware resource conflicts).
  • 4. What is a data hazard? Give one example.
    A data hazard occurs when instructions depend on the results of previous instructions. Example: If instruction 2 needs the result of instruction 1, but instruction 1 hasn't finished yet.
  • 5. What is a multiprocessor system?
    A computer system with two or more CPUs working together, sharing memory and I/O, to run multiple tasks or split one job into several parts.
  • 6. What is the difference between UMA and NUMA memory organizations?
    UMA (Uniform Memory Access): All CPUs access memory with the same speed. NUMA (Non-Uniform Memory Access): Each CPU has local memory (faster) and remote memory (slower to access).
  • 7. Name two common interconnection network topologies in multiprocessors.
    Bus (all CPUs share a single path) and Mesh (CPUs arranged in a grid, each connected to neighbors).
  • 8. What is cache coherence and why is it important?
    Cache coherence means all CPUs have the latest, correct version of shared data in their caches. It's important to avoid errors when different CPUs work on the same memory locations.
  • 9. Briefly explain the write-through and write-back cache protocols.
    Write-through: Data is written to both cache and main memory immediately. Write-back: Data is written to cache first, and main memory is updated later.
  • 10. What is a snoopy cache protocol?
    A method where each cache monitors (snoops) the bus to track changes made by other CPUs, keeping cache data coherent in small/bus-based multiprocessor systems.
