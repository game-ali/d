Questions

  1. What are the three (3) basic computer instruction formats?
  2. What is opcode (operation code) and operand?
  3. What are the different fields of instructions?

Solutions

1. Basic Computer Instruction Formats
  • Memory-Reference Instructions:
    These involve an opcode and a memory address field. They perform operations like loading from memory to register or storing from register to memory. Addressing modes (direct/indirect) are supported. Typical format: opcode + addressing mode bit + memory address.
  • Register-Reference Instructions:
    Operate directly on registers (not memory). Format includes an opcode for a register operation, and bits indicating the specific operation (e.g., complement, increment, clear a register).
  • Input/Output Instructions:
    Used for communication with external devices. Format specifies opcode for I/O and bits to identify device or operation. Distinct from memory/register instructions.
2. Opcode and Operand
  • Opcode (Operation Code): The part of a machine instruction that tells the CPU what operation to perform (e.g., add, subtract, load, jump).
  • Operand: The data or address the opcode works on. Can be constants, variables, memory addresses, or registers.
    Example: In ADD AX, BX, ADD is the opcode, AX and BX are operands.
3. Fields of Instructions
  • Opcode Field: Specifies the operation to perform (add, load, store, jump, etc.).
  • Operand/Address Field(s): Shows the data or location (register or memory) for the operation. Could be one or more fields.
  • Addressing Mode Field: Tells how to interpret the operand fields (e.g., direct, indirect, immediate, register).
  • Additional Fields (Optional):
    • Register specifier fields (which registers involved)
    • Instruction format/extension fields (sub-operations/variants)
    • Condition or flag fields (for conditional instructions)
