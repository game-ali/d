1. Basic Computer Units & Their Functions

  • Functional Units: Input, Memory, ALU (Arithmetic and Logic Unit), Output, Control Unit, Interconnection Network.
  • Input Unit: Gets data from outside (keyboard, mouse) and sends it to computer memory or processor.
  • Memory: Stores data and instructions for processing or later use.
  • Program: Set of instructions in memory that tell the computer what to do with data.
  • Output Unit: Sends results to outside devices (monitor, printer, etc).
  • Control Unit: Directs and coordinates all operations by sending control signals.
  • Interconnection Network: Lets all units exchange data and signals (like a highway for data).
  • Primary Memory vs Secondary Storage: Primary memory (RAM) is fast and temporary; secondary storage (like hard disks) is slower and permanent.
  • Cache Memory: Small, very fast memory close to the CPU to speed up processing of frequently used data.
  • ALU: Does all the math (add, subtract) and logic (compare, AND, OR) operations.
  • Arithmetic Operations: CPU fetches data from memory, processes with ALU, and stores result back in registers or memory.
  • Instruction Register (IR): Holds the current instruction being executed.
  • Load Instruction Steps:
    • Fetch instruction from memory
    • Decode the operation
    • Fetch operand from memory location
    • Store operand in register
  • Add Instruction: Adds contents of two registers and stores the result in another register.
  • Processor-Memory Communication: Uses control signals (read/write) and addresses to move data between CPU and memory.

2. CPU Registers & Data Flow Basics

  • Program Counter (PC): Holds address of the next instruction to execute.
  • Instruction Register (IR): Holds the instruction currently being executed.
  • General-Purpose Registers: Store temporary data and calculation results.
  • Memory Address Register (MAR): Holds address for memory read/write.
  • Memory Data Register (MDR): Holds data being transferred to or from memory.
  • Accumulator: Register used by ALU to store input and results of operations.
  • Processor-Memory Interface: Controls transfer of data between CPU and memory.
  • Program Execution Cycle:
    • Load program into memory
    • Set PC to first instruction
    • Fetch instruction
    • Decode & execute instruction
    • Fetch operands, do operation, store results
    • Repeat for next instruction
  • Interrupt: A signal that makes the CPU stop current work to handle an urgent task.
  • Von Neumann Architecture: Uses same memory for data and instructions. Components: CPU, memory, buses, I/O.
  • Von Neumann CPU Parts: Control Unit, ALU, Registers (PC, IR, AC, MAR, MDR).
  • System Buses:
    • Data Bus: Carries data
    • Address Bus: Carries addresses
    • Control Bus: Carries control signals
  • Control Unit: Coordinates all CPU activities with control signals.
  • I/O Devices: Send and receive data between CPU and outside world.
  • Primary Memory Types: RAM (temporary, fast), ROM (permanent, for boot programs).
